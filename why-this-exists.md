# Why this exists

Every API surface accumulates a documentation debt the moment it has two entries, and the hand-maintained list at the api root was that debt made visible: a JSON file that was accurate on the day it was written and quietly wrong by the second deploy after. The failure mode is not dramatic. Nobody notices a stale registry until they trust it, and by then the cost has moved downstream to whoever built on the missing entry.

The usual fix is discipline: update the list when you deploy. That fix fails for a structural reason, not a moral one. Documentation maintained by memory competes with everything else memory does, and it loses on exactly the busy days when the estate changes most. The durable fix is to remove the human from the loop entirely: the account itself already knows every Worker it runs, each Worker's route already encodes its public URL, and a convention (`/_meta`) lets each Worker answer for its own endpoints. Discovery, probing, and publishing become a cron job, and the registry becomes something the estate emits rather than something anyone writes.

The `/_meta` convention is the half of this that compounds. A registry that scrapes names from an API can tell you a Worker exists; only the Worker can tell you what it is for, and it should say so in a machine-readable shape at a known path, versioned with the code that makes it true. Self-description that deploys with the service cannot drift from the service. The registry then doubles as an adoption dashboard: Workers without `/_meta` are listed as discovered but undocumented, which is both the honest state and a standing, automatically-maintained todo list.

Self-healing is designed in rather than hoped for. The snapshot carries a TTL slightly longer than the rebuild interval, so a missed cron degrades nothing and a dead cron surfaces within the hour. A cold read rebuilds live and re-persists. New Workers are announced by diffing against the previous snapshot, so the system reports change, not state, and stays quiet otherwise. Each choice is small; together they mean the registry needs an operator approximately never.

The transferable principle: any document that describes running systems should be generated by asking the systems, on a schedule, with the deltas reported. If keeping a document true requires a human to remember, the document is already false; the only question is whether anyone has noticed yet.
